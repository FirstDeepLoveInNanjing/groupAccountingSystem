# 功能改造合并说明

本文档合并以下三份说明：

- `举报流程改造说明.md`
- `短信认证服务接入说明.md`
- `短信手机号输入框合并说明.md`

## 一、举报流程改造

### 1. 用户发布举报流程

用户发布举报时流程调整为：

1. 先从当前用户参与的项目中选择项目。
2. 再输入被举报人的用户ID、手机号或邮箱进行精确查找。
3. 填写举报理由后确认提交。
4. 提交时后端会校验举报者和被举报者是否都属于该项目，只有同时属于该项目才会写入举报。

### 2. 举报数据字段

举报数据增加以下字段：

- `accusedUserID`：被举报用户ID。
- `punishmentType`：处罚类型，支持警告、封号1天、封号7天、封号365天。
- `punishmentEndTime`：封号截止时间。

### 3. 管理员举报页面

管理员举报页面已合并：

- `/findReport` 现在同时支持查找和审核举报。
- 管理员可按举报ID、举报者ID、被举报者ID、手机号、邮箱、用户名、项目ID、举报内容搜索。
- 待审核举报可直接在同一页点击“审核举报”。
- 选择“通过”时需要选择处罚，选择“驳回”时不会保存处罚。

### 4. 处罚结果展示

- 举报者在“查看举报状态”中可以看到处罚结果。
- 被警告用户登录后，在首页右上角看到“您在【项目名】中有不合规行为，请注意规范使用本软件”。
- 被封号用户在封号截止前不能通过密码或短信登录，登录页会显示“您在【项目名】中有不合规行为，已被封号，截止到xx年xx月xx日xx时xx分”。

## 二、阿里云短信认证服务接入

本次接入的是阿里云号码认证服务 Dypnsapi 的短信认证服务，不是普通短信服务 Dysmsapi 的短信验证码能力。

已完成：

- 普通用户手机号短信验证码登录。
- 找回密码时使用手机号短信验证码验证身份。
- 注册流程保持不变。
- 邮箱找回密码暂不启用。
- 只允许已注册手机号发送登录/找回密码验证码。

使用的阿里云接口：

- `SendSmsVerifyCode`
- `CheckSmsVerifyCode`

官方文档：

- https://next.api.aliyun.com/document/Dypnsapi/2017-05-25/SendSmsVerifyCode
- https://next.api.aliyun.com/document/Dypnsapi/2017-05-25/CheckSmsVerifyCode

## 三、短信手机号输入框合并

### 1. 原问题

原页面把“发送验证码”和“短信登录/重置密码”拆成了两个独立表单：

- “发送验证码”表单提交到 `/sendSmsLoginCode` 或 `/forgetPassword/sendCode`。
- “短信登录/重置密码”表单提交到 `/ordinaryUserSmsLogin` 或 `/forgetPassword`。

由于两个表单相互独立，每次提交只能带上自己表单内的字段，所以两个表单都放了一个 `phoneNumber` 输入框。

### 2. 当前调整

登录页的“普通用户短信登录”改成一个表单：

- 一行手机号输入框 + “发送验证码”按钮。
- 一行验证码输入框 + “登录”按钮。
- “发送验证码”按钮通过 `formaction="/sendSmsLoginCode"` 提交。
- “登录”按钮使用表单默认地址 `/ordinaryUserSmsLogin`。

找回密码页改成一个表单：

- 一行手机号输入框 + “发送验证码”按钮。
- 一行验证码、新密码、确认新密码 + “重置密码”按钮。
- “发送验证码”按钮通过 `formaction="/forgetPassword/sendCode"` 提交。
- “重置密码”按钮使用表单默认地址 `/forgetPassword`。

页面样式增加 `.form-row`，用于让输入框和按钮横向排列，窄屏时自动换行。

## 五、application.yml 配置

当前配置位置：

```yaml
aliyun:
  sms-auth:
    access-key-id: your-access-key-id
    access-key-secret: your-access-key-secret
    region-id: cn-hangzhou
    country-code: "86"
    sign-name: your-sms-sign-name
    template-code: SMS_334325209
    template-param: '{"code":"##code##","min":"5"}'
    code-length: 6
    valid-time: 300
    duplicate-policy: 1
    interval: 60
    code-type: 1
    case-auth-policy: 1
```

必须替换：

- `access-key-id`：你的 AccessKey ID。
- `access-key-secret`：你的 Access Key Secret。
- `sign-name`：阿里云短信认证服务控制台里的短信签名名。

已按提供的信息配置：

- `template-code: SMS_334325209`

重要说明：

- `template-param` 必须使用 `##code##` 让阿里云生成验证码。
- 不能把验证码写成固定值，例如 `{"code":"123456"}`，否则 `CheckSmsVerifyCode` 无法校验阿里云生成的验证码。
- 当前默认验证码有效期为 `300` 秒，模板参数里的 `min` 是 `5`。

## 六、新增接口

### 1. 页面表单接口

- `POST /sendSmsLoginCode`
  - 参数：`phoneNumber`
  - 作用：发送短信登录验证码。

- `POST /ordinaryUserSmsLogin`
  - 参数：`phoneNumber`, `verifyCode`
  - 作用：校验短信验证码并登录。

- `POST /forgetPassword/sendCode`
  - 参数：`phoneNumber`
  - 作用：发送找回密码验证码。

- `POST /forgetPassword`
  - 参数：`phoneNumber`, `verifyCode`, `newPassword`, `confirmPassword`
  - 作用：校验验证码后重置密码。

### 2. API 接口

- `POST /api/sendSmsLoginCode`
  - 参数：`phoneNumber`

- `POST /api/ordinaryUserSmsLogin`
  - 参数：`phoneNumber`, `verifyCode`

## 七、页面使用流程

### 1. 短信登录

1. 打开 `/signin`。
2. 在“普通用户短信登录”中输入已注册手机号。
3. 点击“发送验证码”。
4. 收到短信后输入验证码。
5. 点击“登录”。

### 2. 找回密码

1. 打开 `/forgetPassword`。
2. 输入已注册手机号并发送验证码。
3. 输入验证码、新密码、确认新密码。
4. 提交后完成密码重置。

### 3. 举报提交

1. 打开举报填写页面。
2. 选择当前用户参与的项目。
3. 输入被举报人的用户ID、手机号或邮箱进行精确查找。
4. 填写举报理由并提交。

### 4. 举报审核

1. 管理员打开 `/findReport`。
2. 使用举报ID、举报者ID、被举报者ID、手机号、邮箱、用户名、项目ID或举报内容搜索。
3. 对待审核举报点击“审核举报”。
4. 选择“通过”时选择处罚，选择“驳回”时不保存处罚。

## 十一、常见问题

如果发送验证码失败，优先检查：

- `sign-name` 是否是短信认证服务控制台中的签名名。
- `template-code` 是否为 `SMS_334325209`。
- 模板变量是否支持 `code` 和 `min`。
- AccessKey 是否有调用 Dypnsapi 的权限。
- 阿里云短信认证服务是否已开通，模板和签名是否已审核通过。

如果校验验证码一直失败，重点检查：

- `template-param` 是否仍为 `{"code":"##code##","min":"5"}`。
- 是否误用了普通短信服务 Dysmsapi 的模板或接口。
- 校验时手机号是否与发码手机号完全一致。
