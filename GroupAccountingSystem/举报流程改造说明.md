# 举报流程改造说明

## 修改内容

1. 用户发布举报流程调整为：
   - 先从当前用户参与的项目中选择项目；
   - 再输入被举报人的用户ID、手机号或邮箱进行精确查找；
   - 填写举报理由后确认提交；
   - 提交时后端会校验举报者和被举报者是否都属于该项目，只有同时属于该项目才会写入举报。

2. 举报数据增加字段：
   - `accusedUserID`：被举报用户ID；
   - `punishmentType`：处罚类型，支持警告、封号1天、封号7天、封号365天；
   - `punishmentEndTime`：封号截止时间。

3. 管理员举报页面合并：
   - `/findReport` 现在同时支持查找和审核举报；
   - 管理员可按举报ID、举报者ID、被举报者ID、手机号、邮箱、用户名、项目ID、举报内容搜索；
   - 待审核举报可直接在同一页点击“审核举报”；
   - 选择“通过”时需要选择处罚，选择“驳回”时不会保存处罚。

4. 处罚结果展示：
   - 举报者在“查看举报状态”中可以看到处罚结果；
   - 被警告用户登录后，在首页右上角看到“您在【项目名】中有不合规行为，请注意规范使用本软件”；
   - 被封号用户在封号截止前不能通过密码或短信登录，登录页会显示“您在【项目名】中有不合规行为，已被封号，截止到xx年xx月xx日xx时xx分”。

## 运行前需要做什么

如果你是重新导入整库，可以直接使用项目根目录外层的 `sys.sql`，里面的 `report` 表结构已经更新。

如果你是在已有数据库上升级，需要先执行：

```sql
source 数据库升级-举报处罚.sql;
```

或者在 MySQL 客户端中打开并执行 `数据库升级-举报处罚.sql`。

注意：升级脚本会给历史举报补一个临时 `accusedUserID = reporterID`，这是为了兼容旧数据；新提交的举报会写入真实被举报人。
