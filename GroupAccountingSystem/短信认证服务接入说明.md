# 阿里云短信认证服务接入说明

## 本次实现内容

本次接入的是阿里云号码认证服务 Dypnsapi 的短信认证服务，不是普通短信服务 Dysmsapi 的短信验证码能力。

已完成：

- 普通用户手机号短信验证码登录。
- 找回密码时使用手机号短信验证码验证身份。
- 注册流程保持不变。
- 邮箱找回密码暂不启用。
- 只允许已注册手机号发送登录/找回密码验证码。

使用的阿里云接口：

- `SendSmsVerifyCode`
- `CheckSmsVerifyCode`

官方文档：

- https://next.api.aliyun.com/document/Dypnsapi/2017-05-25/SendSmsVerifyCode
- https://next.api.aliyun.com/document/Dypnsapi/2017-05-25/CheckSmsVerifyCode

## application.yml 配置

当前配置位置：

```yaml
aliyun:
  sms-auth:
    access-key-id: your-access-key-id
    access-key-secret: your-access-key-secret
    region-id: cn-hangzhou
    country-code: "86"
    sign-name: your-sms-sign-name
    template-code: SMS_334325209
    template-param: '{"code":"##code##","min":"5"}'
    code-length: 6
    valid-time: 300
    duplicate-policy: 1
    interval: 60
    code-type: 1
    case-auth-policy: 1
```

必须替换：

- `access-key-id`：你的 AccessKey ID。
- `access-key-secret`：你的 Access Key Secret。
- `sign-name`：阿里云短信认证服务控制台里的短信签名名。

已按你提供的信息配置：

- `template-code: SMS_334325209`

重要说明：

- `template-param` 必须使用 `##code##` 让阿里云生成验证码。
- 不能把验证码写成固定值，例如 `{"code":"123456"}`，否则 `CheckSmsVerifyCode` 无法校验阿里云生成的验证码。
- 当前默认验证码有效期为 `300` 秒，模板参数里的 `min` 是 `5`。

## 页面使用流程

短信登录：

1. 打开 `/signin`。
2. 在“普通用户短信登录”中输入已注册手机号。
3. 点击“发送验证码”。
4. 收到短信后输入手机号和验证码。
5. 点击“短信登录”。

找回密码：

1. 打开 `/forgetPassword`。
2. 输入已注册手机号并发送验证码。
3. 输入手机号、验证码、新密码、确认新密码。
4. 提交后完成密码重置。

## 新增接口

页面表单接口：

- `POST /sendSmsLoginCode`
  - 参数：`phoneNumber`
  - 作用：发送短信登录验证码。

- `POST /ordinaryUserSmsLogin`
  - 参数：`phoneNumber`, `verifyCode`
  - 作用：校验短信验证码并登录。

- `POST /forgetPassword/sendCode`
  - 参数：`phoneNumber`
  - 作用：发送找回密码验证码。

- `POST /forgetPassword`
  - 参数：`phoneNumber`, `verifyCode`, `newPassword`, `confirmPassword`
  - 作用：校验验证码后重置密码。

API 接口：

- `POST /api/sendSmsLoginCode`
  - 参数：`phoneNumber`

- `POST /api/ordinaryUserSmsLogin`
  - 参数：`phoneNumber`, `verifyCode`

## 常见问题

如果发送验证码失败，优先检查：

- `sign-name` 是否是短信认证服务控制台中的签名名。
- `template-code` 是否为 `SMS_334325209`。
- 模板变量是否支持 `code` 和 `min`。
- AccessKey 是否有调用 Dypnsapi 的权限。
- 阿里云短信认证服务是否已开通，模板和签名是否已审核通过。

如果校验验证码一直失败，重点检查：

- `template-param` 是否仍为 `{"code":"##code##","min":"5"}`。
- 是否误用了普通短信服务 Dysmsapi 的模板或接口。
- 校验时手机号是否与发码手机号完全一致。
