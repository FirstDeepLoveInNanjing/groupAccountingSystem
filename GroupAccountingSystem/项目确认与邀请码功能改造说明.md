# 项目确认与邀请码功能改造说明

本文档说明本次对团体记账系统的功能逻辑改造内容，以及如何运行改造后的版本。

## 一、本次改造内容

### 1. 用户邀请码

为普通用户新增了邀请码字段 `inviteCode`。

- 用户注册时，系统自动生成一个 8 位邀请码。
- 邀请码由大小写字母和数字组成。
- 邀请码用于项目创建者邀请成员加入项目。
- 个人信息页面新增邀请码展示。
- 个人信息页面增加提示：邀请码用于加入项目，请不要随意暴露给陌生人。

涉及文件：

- `src/main/java/com/groupaccountingsystem/pojo/OrdinaryUser.java`
- `src/main/java/com/groupaccountingsystem/service/impl/OrdinaryUserServiceImpl.java`
- `src/main/resources/mapper/OrdinaryUserMapper.xml`
- `src/main/resources/mapper/UserMapper.xml`
- `src/main/resources/templates/getUserInfo.html`

### 2. 邀请成员方式调整

原来创建项目和添加成员时，可以通过用户搜索选择成员。现在改为只能通过成员的邀请码邀请。

- 创建项目页面：输入成员邀请码后添加邀请成员。
- 添加成员页面：输入成员邀请码添加成员。
- 如果邀请码为空或不存在，会提示错误。
- 如果该成员已在项目中，会提示成员已存在。

涉及文件：

- `src/main/resources/templates/CreateActivity.html`
- `src/main/resources/templates/AddMember.html`
- `src/main/java/com/groupaccountingsystem/service/impl/ActivityServiceImpl.java`

### 3. 项目确认流程调整

原流程中，项目创建后不需要成员确认也可以直接结算。本次改为严格的顺序流程：

1. 创建者创建项目。
2. 创建者发起确认。
3. 成员确认参与。
4. 所有项目成员均确认后，创建者才可以发起结算。
5. 成员付款。

新增项目字段 `confirmInitiated`，用于记录创建者是否已经发起确认。

关键限制：

- 成员在创建者发起确认前点击“确认参与”，提示：`创建者未发起确认`。
- 创建者在还有成员未确认时发起结算，提示：`需要所有项目成员确认才能发起结算`。
- 已结算项目不能重复发起结算。

涉及文件：

- `src/main/java/com/groupaccountingsystem/pojo/GroupActivity.java`
- `src/main/java/com/groupaccountingsystem/service/impl/ActivityServiceImpl.java`
- `src/main/java/com/groupaccountingsystem/service/impl/SettlementServiceImpl.java`
- `src/main/resources/mapper/GroupActivityMapper.xml`
- `src/main/resources/templates/InitConfirmActivity.html`
- `src/main/resources/templates/SearchActivity.html`

### 4. 金额逻辑调整

原来项目创建时填写预算，结算时再填写实际金额。本次改为：

- 创建项目时必须填写实际金额。
- 不再在页面展示预算金额。
- 发起确认前，创建者可以在编辑项目页面修改实际金额。
- 发起确认后，实际金额锁定，编辑页面只展示金额，不允许修改。
- 发起结算时直接显示该项目实际金额，不再允许创建者重新填写。

为了兼容原数据库字段，代码仍会把 `budget` 字段同步设置为 `actualAmount`，但页面和业务逻辑不再使用预算金额。

涉及文件：

- `src/main/resources/templates/CreateActivity.html`
- `src/main/resources/templates/EditActivity.html`
- `src/main/resources/templates/InitSettlement.html`
- `src/main/resources/templates/SearchActivity.html`
- `src/main/resources/templates/DisplayActivity.html`
- `src/main/java/com/groupaccountingsystem/service/impl/ActivityServiceImpl.java`
- `src/main/java/com/groupaccountingsystem/service/impl/SettlementServiceImpl.java`

### 5. 项目管理页面调整

原“查找项目”改名为“项目管理”。

普通用户只能查看自己创建或参与的项目，不能再查询到所有项目。管理员入口仍保留查询全部项目的能力。

项目管理列表新增字段：

- 项目创建者：显示创建者用户名。
- 如果创建者是当前用户，旁边用红字小字标注“自己”。
- 确认状态：
  - 当前用户是创建者时，显示“已发起确认”或“未发起确认”。
  - 当前用户是参与者时，显示“已确认”或“未确认”。

操作栏调整：

- 当前用户是项目创建者时，显示：详情、编辑、删除、添加成员、发起确认、发起结算。
- 当前用户是项目参与者时，显示：详情、确认参与、申请退出。
- 首页删除“确认项目/申请退出”入口，该功能合并到项目管理页面。

另外，普通用户直接访问项目详情或收款详情时，也限制为只能访问自己创建或参与的项目。

涉及文件：

- `src/main/resources/templates/SearchActivity.html`
- `src/main/resources/templates/index.html`
- `src/main/resources/templates/ConfirmActivity.html`
- `src/main/java/com/groupaccountingsystem/controller/SystemController.java`
- `src/main/resources/mapper/GroupActivityMapper.xml`

### 6. 项目详情权限调整

项目详情页面中以下操作只对项目创建者显示：

- 编辑项目
- 添加成员
- 移除成员
- 处理退出申请
- 发起确认
- 发起结算
- 查看收款情况

参与者只能查看项目详情和成员信息，不能看到创建者管理按钮。

涉及文件：

- `src/main/resources/templates/DisplayActivity.html`
- `src/main/resources/templates/RemoveMember.html`
- `src/main/resources/templates/ProcessWithdrawApplication.html`

### 7. AA 制结算调整

结算时按 AA 制均分，创建者也计入总人数。

例如：

- 1 个创建者 + 2 个参与者
- 项目实际金额为 `tot`
- 总人数为 3
- 每个参与者需要向创建者支付 `tot / 3`
- 创建者自己的份额不生成付款记录

金额保留两位小数，采用四舍五入。

涉及文件：

- `src/main/java/com/groupaccountingsystem/service/impl/SettlementServiceImpl.java`

## 二、数据库升级

本次新增了数据库升级脚本：

```text
数据库升级-邀请码与项目确认.sql
```

位置：

```text
GroupAccountingSystem/数据库升级-邀请码与项目确认.sql
```

运行新版本前，需要先在当前业务数据库中执行该脚本。

脚本主要做了以下事情：

- 给 `user` 表新增 `inviteCode` 字段。
- 给已有普通用户补齐邀请码。
- 给 `inviteCode` 增加唯一索引。
- 给 `groupactivity` 表新增 `confirmInitiated` 字段。
- 将已有项目的 `budget` 同步为 `actualAmount`，兼容旧数据。

如果是全新建库，也需要确保 `user` 表和 `groupactivity` 表包含上述两个新增字段。

## 三、如何运行新版本

### 1. 确认环境

需要具备以下环境：

- JDK 17
- Maven Wrapper，项目已自带 `mvnw.cmd`
- MySQL 数据库
- 数据库连接配置在：

```text
src/main/resources/application.yml
```

### 2. 执行数据库升级脚本

进入 MySQL 客户端或数据库管理工具，选择项目使用的数据库后执行：

```sql
SOURCE D:/大三下/软工课设/团体记账系统/union/now/GroupAccountingSystem/数据库升级-邀请码与项目确认.sql;
```

如果使用 Navicat、DataGrip、MySQL Workbench 等工具，也可以直接打开该 SQL 文件并执行。

### 3. 编译项目

在项目目录执行：

```powershell
cd D:\大三下\软工课设\团体记账系统\union\now\GroupAccountingSystem
.\mvnw.cmd -DskipTests compile
```

### 4. 运行测试

```powershell
.\mvnw.cmd test
```

本次改造后已验证：

- `.\mvnw.cmd -DskipTests compile` 通过
- `.\mvnw.cmd test` 通过

### 5. 启动系统

```powershell
.\mvnw.cmd spring-boot:run
```

启动后，在浏览器访问：

```text
http://localhost:8080
```

如果 `application.yml` 中配置了其他端口，请以实际配置为准。

## 四、建议测试流程

### 1. 注册与邀请码

1. 注册一个普通用户。
2. 登录该用户。
3. 进入“查看个人信息”。
4. 确认页面展示邀请码和安全提示。

### 2. 创建项目与邀请成员

1. 创建者登录。
2. 确保创建者已上传收款码。
3. 进入“创建项目”。
4. 填写项目名称、实际金额。
5. 输入其他用户的邀请码邀请成员。
6. 创建成功后进入“项目管理”查看项目。

### 3. 确认流程

1. 成员在创建者发起确认前点击“确认参与”。
2. 系统应提示：`创建者未发起确认`。
3. 创建者点击“发起确认”。
4. 成员再次点击“确认参与”。
5. 成员状态变为“已确认”。

### 4. 结算流程

1. 创建者在还有成员未确认时点击“发起结算”。
2. 系统应提示：`需要所有项目成员确认才能发起结算`。
3. 所有成员确认后，创建者再次发起结算。
4. 结算页面显示固定的实际金额，不再提供金额输入框。
5. 结算成功后，参与者在“交费结算”中看到付款记录。

### 5. AA 制金额验证

以 1 个创建者和 2 个参与者、实际金额 300 为例：

- 总人数为 3。
- 每个参与者付款金额为 100。
- 创建者不需要给自己生成付款记录。

## 五、注意事项

- 必须先执行数据库升级脚本，否则新字段不存在会导致运行时报错。
- 老数据中的成员确认状态如果仍是旧的乱码值，建议统一修正为：`已确认`、`未确认`、`申请退出`。
- 老项目如果 `actualAmount` 为空，需要补充实际金额后再走新的确认和结算流程。
- 本次未删除数据库中的 `budget` 字段，只是在业务和页面层不再使用，避免破坏旧表结构兼容性。
